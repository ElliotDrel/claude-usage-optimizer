# Security Policy

## Reporting a vulnerability

Do not open a public GitHub issue for security-sensitive reports.

Send the report privately to the repository owner with:

- a clear description of the issue
- affected files or routes
- reproduction steps
- impact assessment
- any suggested remediation

## Scope notes

Security-sensitive areas in this repo include:

- setup and secret-writing flows
- local environment and credential handling
- scheduler and sender execution paths
- deployment scripts and service configuration

## Response expectations

Best-effort triage:

- initial acknowledgment within 7 days
- remediation timing based on severity and reproducibility

If a report cannot be reproduced, it may be closed with request for more detail.
