# Quick Task 260406-ga0: Build Claude.ai Usage Tracker - Plan

**Created:** 2026-04-06
**Status:** Ready for execution

## Objective

Rewrite the existing Claude.ai usage tracker to match agreed specs:
- SQLite storage (not JSONL)
- Next.js + Recharts dashboard (not vanilla HTML)
- Correct endpoint + 5-minute polling interval
- Proper project structure with npm dependencies

## Decisions (Locked)

- Direct API call to `https://claude.ai/api/organizations/{orgId}/usage`
- Session cookie authentication
- 5-minute fixed polling interval
- SQLite via better-sqlite3, WAL mode, keep forever
- Next.js + Recharts local dashboard
- Web usage only for v1

## Tasks

### Task 1: Project setup + dependencies
**Files:** package.json, next.config.mjs, .env.example, .gitignore, tsconfig.json
**Action:**
- Initialize Next.js project with TypeScript
- Add dependencies: better-sqlite3, @types/better-sqlite3, recharts, date-fns

- Update .env.example with correct defaults (5-min interval, correct endpoint format)
- Update .gitignore for Next.js + data directory

### Task 2: Collector + SQLite storage (backend core)
**Files:** src/lib/db.ts, src/lib/collector.ts, src/lib/config.ts, src/lib/normalize.ts
**Action:**
- `db.ts`: SQLite setup with WAL mode, schema creation (usage_snapshots table with timestamp index), insert/query functions
- `collector.ts`: UsageCollector class with 5-min setInterval polling, cookie/bearer auth, error logging + retry-on-next-interval
- `config.ts`: Load from .env (endpoint, auth, poll interval, data dir)
- `normalize.ts`: Normalize API response, handle unknown keys, store raw JSON
**Schema:**
```sql
CREATE TABLE usage_snapshots (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  timestamp TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'ok',
  five_hour_utilization REAL,
  five_hour_resets_at TEXT,
  seven_day_utilization REAL,
  seven_day_resets_at TEXT,
  raw_json TEXT,
  error_message TEXT
);
CREATE INDEX idx_snapshots_timestamp ON usage_snapshots(timestamp);
```

### Task 3: API routes
**Files:** src/app/api/dashboard/route.ts, src/app/api/poll/route.ts, src/app/api/snapshots/route.ts
**Action:**
- GET /api/dashboard: Query SQLite, build dashboard data (health, timeline, peak hours, heatmap)
- POST /api/poll: Trigger immediate poll, return snapshot
- GET /api/snapshots: Return raw snapshot data with optional date range filtering

### Task 4: Dashboard UI with Recharts

**Action:**

- UsageCards: Current utilization for each window with color thresholds (green <50%, yellow 50-80%, red >80%)
- UsageTimeline: Recharts AreaChart for utilization over time, multi-series

- Heatmap: 7x24 grid (day-of-week x hour) with color intensity
- CollectorHealth: Status pill, auth mode, poll interval, failure count
- ExtraUsage: Display any non-standard API response fields
- Poll Now button triggers immediate collection + UI refresh
