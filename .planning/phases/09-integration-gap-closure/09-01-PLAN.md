<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Middleware | Unauthenticated user visits any route; middleware checks setup_complete without additional auth (safe because dashboard is bound to 127.0.0.1) |
| Middleware → app_meta | Middleware reads setup_complete flag from SQLite; database is local and not subject to network interception |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Authorization | setup gate bypass | mitigate | Middleware is on critical path before any handler executes; no other route can be reached without passing the gate. No alternate entry points. |
| T-09-02 | Denial of Service | repeated DB reads on middleware | accept | Each middleware invocation reads app_meta once synchronously; no unbounded loops or retries. Normal request volume will not exhaust resources. |
| T-09-03 | Information Disclosure | error messages leak setup state | mitigate | Error handling in proxy.ts (catch block) conservatively redirects to /setup without exposing error details. |

</threat_model>
