```typescript
export async function proxy(request: NextRequest) {
  const pathname = request.nextUrl.pathname
  // Skip setup check for /setup and /api/setup
  if (pathname.startsWith('/setup') || pathname.startsWith('/api/setup')) {
    return NextResponse.next()
  }
  
  try {
    const config = getConfig()
    const meta = getAppMeta(config)
    const setupComplete = meta.get('setup_complete')
    
    if (setupComplete !== 'true') {
      return NextResponse.redirect(new URL('/setup', request.url))
    }
  } catch (err) {
    console.warn('[proxy] setup check failed:', err)
    if (!pathname.startsWith('/setup')) {
      return NextResponse.redirect(new URL('/setup', request.url))
    }
  }
  
  return NextResponse.next()
}

export const config = {
  matcher: [
    '/((?!_next|setup|api/setup|.*\\.png$|.*\\.svg$).*)',
  ],
}
```

- Third-party: NextResponse, NextRequest from next/server
- Internal: getConfig from @/lib/config, getAppMeta from @/lib/db

**Naming:**
- Function: `proxy` (Next.js v16 convention for proxy.ts exports)
- Export: named `proxy` + named `config`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → API | User submits form with auth credentials; HTTPS used (enforced by systemd binding to 127.0.0.1, SSH tunnel recommended) |
| API → staging file | Secrets written to disk briefly; staging file is mode 640, deleted after merge |
| API → sudo helper | Invokes external script using execFileNoThrow (safe, no shell); no user input in command, only fixed path |
| sudo helper → /etc/ | Privilege escalation to write /etc/claude-sender.env; helper accepts no arguments |
| middleware → setup state | Checks app_meta.setup_complete from DB; redirects to /setup if not 'true'; gate enforced before dashboard render |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering / Shell Injection | execFileNoThrow invoked for sudo | mitigate | Use execFileNoThrow with array args (never exec with string interpolation); no secrets in args; all config from staging file |
| T-07-02 | Information Disclosure | Staging file written to /opt/data/ | mitigate | Mode 0o640 (not world-readable); owned by claude-tracker; deleted after merge |
| T-07-03 | Elevation of Privilege | Sudo helper escalates to write /etc/ | mitigate | Entry is NOPASSWD but narrowly scoped to single script; helper accepts no args; fixed path only |
| T-07-04 | Denial of Service | Sudo helper fails, staging file left behind | mitigate | Error handler wraps in try/catch, cleans up staging file on failure |
| T-07-05 | Information Disclosure | OAuth token visible in logs or error messages | mitigate | Don't log token values; return generic error messages to API caller; no token in response JSON |
| T-07-06 | Elevation of Privilege | /setup accessible after setup_complete='true' | mitigate | Middleware (Task 1) checks setup_complete state via app_meta and redirects /setup to /dashboard if already complete |
</threat_model>
