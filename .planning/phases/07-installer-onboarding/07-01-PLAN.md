<objective>
Create a single bash installer script (`scripts/install.sh`) that can be invoked via `curl ... | bash` and fully provisions a fresh Ubuntu 22.04 e2-micro VM with all system-level dependencies, swap, Node.js 20, the app (cloned/built), systemd unit, and sudoers entry for privilege escalation — in a completely idempotent manner so re-running is always safe.

**Purpose:** Enable a non-technical user to go from "I have an Ubuntu 22.04 VM" to "running app on 127.0.0.1:3018" with a single command, no SSH loops or manual steps.

**Output:** Executable `scripts/install.sh` committed to the repo; referenced as a raw GitHub URL in Phase 7 HOSTING-STRATEGY.md updated installation steps.
</objective>

  <action>
Create `scripts/install.sh` (755-mode executable) that fully provisions a fresh Ubuntu 22.04 e2-micro VM. The script MUST:

1. **Set strict mode** — `set -e` and `set -u` to exit on errors and undefined vars. Trap errors with line number reporting.
2. **Verify root** — Exit with error if not running as root or via sudo.
3. **Update package lists** — `apt-get update -qq` (quiet to reduce noise).
4. **Install system packages** — `apt-get install -y git curl sqlite3` in a single invocation.
5. **Provision 2 GB swap (idempotent):**
   - Guard: `[ ! -f /swapfile ]` before creating
   - If not present: `dd if=/dev/zero of=/swapfile bs=1G count=2`, `chmod 600 /swapfile`, `mkswap /swapfile`, `swapon /swapfile`, append to `/etc/fstab` once
   - If present: log "already exists" and skip
6. **Install Node.js 20 via NodeSource (idempotent):**
   - Guard: Check if Node.js 20 is already installed via `command -v node` and version check
   - If not present: `curl -fsSL https://deb.nodesource.com/setup_20.x | bash -` then `apt-get install -y nodejs`
   - If present: log the existing version
7. **Create service user `claude-tracker` (idempotent):**
   - Guard: `id -u claude-tracker` to check if user exists
   - If not present: `useradd --system --home /nonexistent --shell /bin/false claude-tracker`
   - If present: log "already exists"
8. **Clone or pull repo to `/opt/claude-usage-optimizer` (idempotent):**
   - If dir does not exist: `git clone https://github.com/elliotdrel/claude-usage-optimizer.git /opt/claude-usage-optimizer`
   - If dir exists: `cd /opt/claude-usage-optimizer && git pull origin main`
9. **Build the app:**
   - `cd /opt/claude-usage-optimizer && npm ci --omit=dev && npm run build`
10. **Set up data directory (owned by claude-tracker user):**
    - `mkdir -p /opt/claude-usage-optimizer/data`
    - `chown claude-tracker:claude-tracker /opt/claude-usage-optimizer/data`
    - `chmod 750 /opt/claude-usage-optimizer/data`
11. **Pre-create `/etc/claude-sender.env` with placeholder values (mode 600):**
    - Write env file with empty placeholders for `CLAUDE_CODE_OAUTH_TOKEN`, `CLAUDE_SESSION_COOKIE`, `CLAUDE_BEARER_TOKEN`, `user_timezone=America/Los_Angeles`, `GCS_BACKUP_BUCKET=`
    - `chmod 600 /etc/claude-sender.env` and `chown root:root`
12. **Install sudoers entry at `/etc/sudoers.d/claude-tracker`:**
    - Write single line: `claude-tracker ALL=(ALL) NOPASSWD: /opt/claude-usage-optimizer/scripts/write-env.sh`
    - `chmod 440 /etc/sudoers.d/claude-tracker` (sudoers requires 440 or 400, not 644)
13. **Copy systemd unit file and enable service:**
    - Copy `/opt/claude-usage-optimizer/claude-tracker.service` to `/etc/systemd/system/claude-tracker.service`
    - `systemctl daemon-reload` and `systemctl enable claude-tracker.service`
14. **Start the service:**
    - `systemctl start claude-tracker.service`
15. **Exit with success message:**
    - Print clear message: "Installation complete. Service is starting. Access the setup wizard at http://127.0.0.1:3018"
    - `exit 0`

**Security notes (per RESEARCH.md threat register):**
- No hardcoded secrets in the script (secrets come from wizard form post-install, written to staging file)
- All variable expansions are inside double quotes to prevent word splitting: `"$REPO_DIR"`, `"$CLAUDE_TRACKER"`
- Absolute paths only — no relative paths or user input interpolation into shell commands
- `set -u` catches typos in variable names

**Tested path:** The script must work when invoked as `bash <(curl -sL https://raw.githubusercontent.com/elliotdrel/claude-usage-optimizer/main/scripts/install.sh)` or `bash ./scripts/install.sh` locally.

**Per D-01, D-02, D-03:** This script is the full automation of HOSTING-STRATEGY.md Step 4 (packages through systemd unit install). GCS bucket creation (Step 9) remains manual.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| installer → system | Non-interactive bootstrap; runs as root via `curl \| bash`; elevated privileges required for systemd, /etc/, swap |
| installer → GitHub | Fetches repo and NodeSource scripts; relies on HTTPS and DNS resolution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | Installer script sourced from GitHub | mitigate | HTTPS enforced; use raw.githubusercontent.com URL pinned in documentation; verify on local test before release |
| T-07-02 | Elevation of Privilege | Installer runs as root | mitigate | Verify idempotency on re-run; no `sudo` escalation inside script (runs as root already) |
| T-07-03 | Information Disclosure | Env file pre-created with empty placeholders | mitigate | Mode 600 and root:root ownership; no secrets written during provisioning; wizard fills secrets post-install |
| T-07-04 | Denial of Service | Swap provisioning or package install fails on re-run | mitigate | Idempotency guards on every system-level op (swap file check, user existence check, Node version check) |
| T-07-05 | Elevation of Privilege | Sudoers entry installed without safeguards | mitigate | Entry is narrowly scoped: `claude-tracker ALL=(ALL) NOPASSWD: /opt/.../scripts/write-env.sh` only (no wildcards, no ALL commands) |
</threat_model>
