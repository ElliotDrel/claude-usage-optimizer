    4. **send() function** (main export):
       ```typescript
       export async function send(
         config: Config,
         opts?: {
           timeoutMs?: number;
           scheduledFor?: string | null;
           isAnchor?: number;
         }
       ): Promise<SendLogRow> {
         const timeoutMs = opts?.timeoutMs ?? 60_000;
         const scheduledFor = opts?.scheduledFor ?? null;
         const isAnchor = opts?.isAnchor ?? 0;
         const startTime = Date.now();
         const question = pickQuestion();

         return new Promise((resolve) => {
           const cwd = os.tmpdir(); // D-06: spawn from isolated temp dir
           const child = spawn("claude", ["-p", question, "--model", "haiku"], {
             cwd,
             stdio: ["ignore", "pipe", "pipe"],
           });

           let stdout = "";
           let stderr = "";
           let finished = false; // Guard against race condition (Pitfall 2)

           child.stdout?.on("data", (data) => {
             stdout += data.toString();
           });

           child.stderr?.on("data", (data) => {
             stderr += data.toString();
           });

           const timer = setTimeout(() => {
             child.kill("SIGTERM");
           }, timeoutMs);

           child.on("error", (err) => {
             clearTimeout(timer);
             if (finished) return;
             finished = true;

             const duration = Date.now() - startTime;
             const msg = err instanceof Error ? err.message : String(err);
             console.error("[sender]", msg);

             insertSendLog(config, {
               fired_at: new Date().toISOString(),
               scheduled_for: scheduledFor,
               is_anchor: isAnchor,
               status: "error",
               duration_ms: duration,
               question,
               response_excerpt: null,
               error_message: msg,
             });

             resolve({
               id: 0, // Will be populated by insertSendLog
               fired_at: new Date().toISOString(),
               scheduled_for: scheduledFor,
               is_anchor: isAnchor,
               status: "error",
               duration_ms: duration,
               question,
               response_excerpt: null,
               error_message: msg,
             });
           });

           child.on("exit", (code, signal) => {
             clearTimeout(timer);
             if (finished) return;
             finished = true;

             const duration = Date.now() - startTime;
             let status: string;
             let responseExcerpt: string | null = null;
             let errorMessage: string | null = null;

             if (signal === "SIGTERM" || duration >= timeoutMs) {
               status = "timeout";
               errorMessage = `Timeout after ${timeoutMs}ms`;
             } else if (code === 0) {
               status = "ok";
               responseExcerpt = stdout.slice(0, 500);
             } else {
               status = "error";
               errorMessage = stderr || `Exit code ${code}`;
             }

             const row = insertSendLog(config, {
               fired_at: new Date().toISOString(),
               scheduled_for: scheduledFor,
               is_anchor: isAnchor,
               status,
               duration_ms: duration,
               question,
               response_excerpt: responseExcerpt,
               error_message: errorMessage,
             });

             resolve(row);
           });
         });
       }
       ```

    5. **Export QUESTIONS** (D-02: make available for testing and reference):

    **Key design principles:**
    - Always spawn from os.tmpdir() (D-06: prevents CLAUDE.md context leakage per Pitfall 1)
    - Use array form spawn([cmd], [args], {cwd}) — never shell: true (security: prevents injection)
    - Guard against timeout race condition with `finished` flag (Pitfall 2)
    - Cap response excerpt at 500 chars to avoid memory exhaustion
    - Return full SendLogRow with id from insertSendLog()
    - No retry logic (D-01: explicitly excluded)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI spawning from untrusted input | Question constant is hardcoded (no user input); args passed to spawn are always ["−p", question, "--model", "haiku"] |
| Process isolation | Spawn runs from os.tmpdir(), not project root (prevents CLAUDE.md context leakage per D-06) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | S/T (Spoofing / Tampering) | Shell injection via spawn() | mitigate | Always use array form spawn(cmd, [args], {cwd}), never shell: true. Question is hardcoded constant, no user input. Verified in code: no string interpolation in spawn args. |
| T-03-05 | I (Information Disclosure) | Project CLAUDE.md context leakage | mitigate | Spawn from os.tmpdir() (D-06), not process.cwd(). Prevents claude CLI from reading project .claude/CLAUDE.md which could alter auth or leak context. |
| T-03-06 | D (Denial of Service) | Unbounded stdout buffering | mitigate | Cap response_excerpt at 500 chars. Do not buffer entire stdout indefinitely. |
| T-03-07 | D (Denial of Service) | Zombie processes on timeout | mitigate | Use finished flag guard + SIGTERM timeout. Phase 6 monitoring can add stall detection. |

</threat_model>
