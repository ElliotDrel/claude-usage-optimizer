3. **Modify runTick() function** (lines 298-401):
   
   **At the very start of runTick(), before ANY other logic:**
   Add unconditional timestamp write:
   ```typescript
   // Unconditionally write timestamp on every tick (stall detection depends on this)
   writeMeta(db, "last_tick_at", nowFn().toISOString());
   ```
   This is CRITICAL — every tick must write the timestamp, even if nothing else happens (pause check passes, no sends due, no recompute). The stall detector depends on this to distinguish between "app is paused" and "app is hung."

   **After the pause check (after the `if (paused === "true") return;` line):**
   Add stall detection watchdog:
   ```typescript
   // Check if more than 5 minutes since last tick (stall detection per D-06)
   const lastTickAtStr = readMeta(db, "last_tick_at");
   if (lastTickAtStr && lastTickAtStr !== nowFn().toISOString()) {
     const lastTick = new Date(lastTickAtStr);
     const elapsedSeconds = (nowFn().getTime() - lastTick.getTime()) / 1000;
     if (elapsedSeconds > 300) { // 5 minutes = 300 seconds
       console.error(`[scheduler] STALL DETECTED: ${elapsedSeconds}s since last tick`);
       void postDiscordNotification(
         "Scheduler Stall",
         `No scheduler tick recorded for ${Math.floor(elapsedSeconds)}s (threshold: 300s)`,
         nowFn()
       );
     }
   }
   ```
   **Explanation:** Read the last_tick_at value written at the start of this tick. If it differs from the current time by >300 seconds, a previous tick was delayed/missed. Fire a webhook notification asynchronously.

4. **Rationale for placement:**
   - Timestamp write at TOP: Essential for stall detection; must happen before pause check
   - Stall check after pause: Paused scheduler will still write ticks, so stall detection is still valid; stall check only fires if ticks are genuinely missing (not just paused)
   - Both operations are read/write from app_meta using existing helpers (readMeta, writeMeta)

**Import statement location:** Add `import { postDiscordNotification } from "./notifier";` after the existing imports (after line 17).

**Test consideration:** The stall detection logic should work with the fake-clock testing already in place for the scheduler. The 300-second threshold is hard-coded per D-06.

| T-06-10 | Denial of Service | False stall alerts due to pause or legitimate slow tick | mitigate | Stall detection threshold is 300 seconds (5 minutes); scheduler ticks every 60 seconds.
