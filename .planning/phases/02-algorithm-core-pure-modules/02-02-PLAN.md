```typescript
export interface FireTime {
  hour: number;           // 0–23 user-local
  minute: number;         // 0–59
  isAnchor: boolean;
  jitterMinutes: number;  // always 0 for anchor; 0–5 (integer) for non-anchors

export interface ScheduleOptions {
  anchorOffsetMinutes?: number;       // default: 5
  defaultSeedTime?: string;           // default: "05:05"  format "HH:MM"
  overrideStartTime?: string | null;  // format "HH:MM"; if non-null/non-empty, short-circuits peak detection
}
```

**Exported function:**
```typescript
export function generateSchedule(
  peakBlock: PeakBlock | null,
  options: ScheduleOptions = {}
): FireTime[]
```

**Algorithm (implement exactly):**

1. Resolve the anchor `{ hour: number, minute: number }`:

   a. If `options.overrideStartTime` is non-null and non-empty string → parse `"HH:MM"` into `{ hour, minute }`. This is the anchor exactly as-is (the override already encodes the user's desired start time; do NOT add `anchorOffsetMinutes` on top of an override).

   b. Else if `peakBlock` is non-null → `anchorHour = peakBlock.midpoint`, `anchorMinute = options.anchorOffsetMinutes ?? 5`. Anchor = `{ hour: anchorHour % 24, minute: anchorMinute }`.

   c. Else (peakBlock is null, no override) → parse `options.defaultSeedTime ?? "05:05"` into `{ hour, minute }`. Use this directly as the anchor (the default seed already includes the `:05` offset per the design spec).

2. Build the chain of 5 slots. For slot index `n` (0..4):
   - `totalMinutes = anchorHour * 60 + anchorMinute + n * 5 * 60`
   - `slotHour = Math.floor(totalMinutes / 60) % 24`
   - `slotMinute = totalMinutes % 60`
   - `isAnchor = n === 0`
   - `jitterMinutes = isAnchor ? 0 : Math.floor(Math.random() * 6)` (0–5 inclusive, using `Math.floor(Math.random() * 6)`)

3. Return the array of 5 `FireTime` objects.

**Helper (internal, not exported):**
```typescript
function parseHHMM(s: string): { hour: number; minute: number } {
  const [h, m] = s.split(":").map(Number);
  return { hour: h ?? 0, minute: m ?? 0 };
}
```

**Named exports only. No default export. `function` keyword for all top-level exports.**

Note on override anchor minute: When the override is `"14:30"`, the anchor is `{ hour: 14, minute: 30, isAnchor: true, jitterMinutes: 0 }`. The chain then proceeds at 19:30, 00:30, 05:30, 10:30. The `:05` offset is a peak-detection artifact baked into `anchorOffsetMinutes`; overrides bypass it by design (SCHED-09 / D-03).

```typescript
describe("generateSchedule — 5-hour spacing", () => {
```
