**Algorithm (implement exactly as described):**

1. Filter snapshots to `status === "ok"`. Sort by `timestamp` ascending.
2. Count distinct calendar days (YYYY-MM-DD in user-local time via `Intl.DateTimeFormat`). If distinct days < 3, return `null`.
3. Build a `hourlyDelta: number[]` array of length 24, initialized to 0.
4. Iterate pairwise (i=1..N): for each consecutive pair `(prev, curr)`, compute the delta using `computeUsageDelta(prev.five_hour_utilization, curr.five_hour_utilization, prev.five_hour_resets_at, curr.five_hour_resets_at)`. Get the user-local hour of `curr.timestamp` via `Intl.DateTimeFormat` with the given timezone. Accumulate delta into `hourlyDelta[localHour]`.
5. Slide a 4-hour window across all 24 starting positions (wrapping via `% 24`). For each start index `s` (0..23), sum `hourlyDelta[(s+0)%24] + hourlyDelta[(s+1)%24] + hourlyDelta[(s+2)%24] + hourlyDelta[(s+3)%24]`.
6. Find the maximum window sum. Collect all windows tied at that maximum.
7. **Tiebreak:** Among tied windows, pick the one whose midpoint `(s+2)%24` is closest to hour 12 (i.e., minimize `Math.abs(mid - 12)`). If still tied, pick the earliest startHour (smallest `s`).
8. Return `{ peakBlock: { startHour: s, endHour: (s+4)%24, sumDelta, midpoint: (s+2)%24 }, midpoint: (s+2)%24 }`.
